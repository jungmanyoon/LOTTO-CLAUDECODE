# 📊 통합 확률 기반 적응형 필터링 시스템 가이드

## 🎯 핵심 개념

### 임계값의 의미
**`global_probability_threshold`**: 제외할 패턴의 최대 출현율

- **임계값 1.0%** = "1.0% 이하로 출현하는 패턴을 제외한다"
- **임계값이 높을수록** = 더 많은 패턴을 제외 (공격적)
- **임계값이 낮을수록** = 더 적은 패턴만 제외 (보수적)

## 📈 임계값 설정 가이드

| 임계값 | 성격 | 제외량 | 설명 | 추천 대상 |
|--------|------|--------|------|-----------|
| **0.1%** | 극도로 보수적 | 극소량 | 거의 나오지 않은 패턴만 제외 | 초보자, 안전 우선 |
| **0.5%** | 보수적 | 적음 | 매우 드문 패턴만 제외 | 안정적 운영 |
| **1.0%** | 표준 (권장) | 중간 | 균형잡힌 필터링 | 일반 사용자 |
| **1.5%** | 약간 공격적 | 다소 많음 | 조금 더 많이 제외 | 경험자 |
| **2.0%** | 공격적 | 많음 | 상당량 제외 | 위험 감수 가능 |
| **3.0%** | 매우 공격적 | 매우 많음 | 대량 제외 | 고위험 전략 |

| 홀수 0개 | 0.0% | ✅ 제외 | ✅ 제외 | ✅ 제외 |

- **임계값 2.0%**: 홀수 0개, 6개 제외 (더 많이 제외)

## 💡 전략별 추천 설정

### 1. 초보자/안전 우선 전략
```yaml
global_probability_threshold: 0.5  # 매우 보수적
```
- 극히 드문 패턴만 제외
- 당첨번호 제외 위험 최소화
- 확률 개선 효과는 작음

### 2. 균형잡힌 표준 전략 (권장)
```yaml
global_probability_threshold: 1.0  # 표준
```
- 1% 미만 패턴 제외
- 안전성과 효율성 균형
- 대부분 사용자에게 적합

### 3. 공격적 전략
```yaml
global_probability_threshold: 2.0  # 공격적
```
- 2% 미만 패턴 모두 제외
- 더 많은 조합 제외
- 위험 증가, 효율 증가

## ⚠️ 주의사항

1. **임계값이 높을수록 위험 증가**
   - 2% 이상: 실제 당첨번호도 제외될 가능성 있음
   - 3% 이상: 고위험, 신중한 사용 필요

2. **백테스팅 필수**
   - 임계값 변경 후 반드시 과거 데이터로 검증
   - 실제 당첨번호가 제외되는지 확인

3. **점진적 조정**
   - 처음에는 1.0%로 시작
   - 결과를 보며 0.1%씩 조정

## 🔧 적용 방법

### 1단계: 설정 파일 수정
```yaml
# configs/adaptive_filter_config.yaml
global_probability_threshold: 1.0  # 원하는 값으로 변경
```

### 2단계: 프로그램 실행
```bash
python main.py
```

설정이 자동으로 모든 필터에 적용됩니다.

## 📝 요약

- **임계값 ↑** = 더 많이 제외 (공격적)
- **임계값 ↓** = 더 적게 제외 (보수적)
- **권장값**: 1.0% (균형잡힌 표준값)
- **한 줄 요약**: "임계값% 이하로 나타나는 패턴을 제외한다"
