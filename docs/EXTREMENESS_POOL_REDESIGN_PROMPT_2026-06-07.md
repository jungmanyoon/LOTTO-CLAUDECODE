## Context (carry forward) — 2026-06-07 진단으로 확정된 사실
- 현 극단성 점수 = 마할라노비스거리^2(연속9: sum,std,max_gap,range,odd_count,low_count,ac_value,prime_count,distinct_last_digits) + 비모수 꼬리페널티(이산4: max_consecutive,odd_count,max_same_last_digit,max_section_occupancy). 점수 상위(극단)부터 잘라 K=1.5M 유지.
- **문제1(실측)**: 제거를 마할라노비스(제거평균10.21 vs 유지4.15, 차 +6.06)가 압도. 희귀패턴 페널티4종 합은 +1.28뿐. → 시스템이 "역사상 드문 것"이 아니라 **"역사 평균에서 먼 것"**을 제거 중 = 사용자 의도와 불일치.
- **문제2(가장자리 과잉제거)**: 전체 제거율 81.6%인데 번호1 포함 89.4% / 45 포함 87.6% / 1·45 둘다 98.5% / 중앙만 77.1%. range·low_count가 비대칭 분포라 가우시안 마할라노비스가 양끝을 "꼬리"로 오인 → 역사상 정상 출현하는 양끝 번호 조합을 과잉 제거.
- **문제3(보존율 ~무작위)**: 역대 당첨번호 풀 보존율 19.8%(243/1227) ≈ 무작위 18.4%(×1.06뿐). 양끝 포함 당첨 18.1% < 미포함 22.4%. → "같은 예산에 당첨영역 더 덮기" 목표가 거의 미작동.

## 구현 설계 (Codex gpt-5.5 + Gemini 3.1-pro 합의)
- **마할라노비스를 핵심 제거기준에서 제외**(원하면 진단용으로만 유지).
- **비모수 꼬리확률 점수로 교체**: 각 특징 f에 대해 역사적 당첨번호의 eCDF로 그 값의 양극단 꼬리확률 `P_f = min(eCDF(x), 1 - eCDF(x))`(이산은 평활 빈도). `score = Σ_f -log(P_f)`. 점수 상위 제거로 K=1.5M 제어.
  - 효과: 역사상 ~3% 나오는 range=44는 생존, 진짜 하위 0.x% 꼬리만 제거 → 양끝 정상조합 보존 + "어느 특징이 역사 하위 몇 %라 제거됐다"는 **투명한 사유** 설명 가능(= 사용자가 원한 명시적 패턴 제외).
- **번호 생존율 가드레일**: 특정 번호(특히 1-5, 41-45)가 전체 평균 대비 과잉 제거되면 경고/완화.
- (선택) 명시적 패턴 카탈로그 + 제거예산 방식 병행(Codex 권장). 1차는 꼬리확률 점수로 시작.

## 검증 프로토콜 (필수 · 정직)
blind walk-forward(fold 마다 train_until 이하만 fit). **신규 스코어러 vs 현행 vs 무작위** 3자 비교:
1. **당첨 보존율** — 목표: 현 19.8% 및 무작위 18.4%를 통계적으로 초과.
2. **가장자리 번호 커버 회복** — 1·45 제거율이 전체 81.6%에 수렴하는지.
3. **포트폴리오 실적** — 5세트/다수세트의 mean best-match, P(3+) (run_pool_selection_backtest 류로) vs 현행/무작위.
4. **과적합 가드** — hold-out 분리 + 효과크기 임계 + (후보 여럿이면) 다중비교 보정.
- **현재보다 통계적으로 나을 때만 채택안 제시. 아니면 정직하게 기각**(직전 '필터 추가'는 음성이었음 - 동일 엄격성 적용).
