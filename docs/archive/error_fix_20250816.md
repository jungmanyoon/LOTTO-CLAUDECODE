- main.py에서 combinations_db의 filter_results 테이블을 조회하려 했으나, 이 테이블은 존재하지 않음
- 실제로는 filtered_combinations 테이블이 존재함

**해결**: main.py 수정
```python
# 수정 전
with db_manager.combinations_db._create_connection() as conn:

    query = """
        SELECT combination, COUNT(*) as pass_count
        FROM filter_results
        WHERE passed = 1
        GROUP BY combination
        ORDER BY pass_count DESC
        LIMIT 100
    """
    cursor.execute(query)
    filtered_combos = cursor.fetchall()
```

### 2. 패턴 기반 번호 생성 unpacking 에러 해결

**원인**:
- lotto_numbers 테이블이 4개 컬럼(round, numbers, draw_date, created_at)을 가지고 있음
- 하지만 코드는 3개 값만 unpacking하려고 시도

**해결**: src/core/specialized_databases.py 수정
```python
# 수정 전
def get_all_numbers(self) -> List[Tuple[int, str, str]]:
    cursor.execute("SELECT * FROM lotto_numbers ORDER BY round")
    return cursor.fetchall()

# 수정 후  
def get_all_numbers(self) -> List[Tuple[int, str, str]]:
    cursor.execute("SELECT round, numbers, draw_date FROM lotto_numbers ORDER BY round")
    return cursor.fetchall()
```
