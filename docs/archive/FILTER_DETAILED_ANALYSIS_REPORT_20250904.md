## 🔧 문제 원인 분석

### 1. match 필터의 문제점
현재 코드를 다시 확인한 결과:
- `max_match = 4`로 설정되어 있음
- 하지만 **과거 1,186개 당첨번호 모두와 비교**
- 실제 4개 이상 일치 확률은 0.11%인데 80.90% 제외는 비정상

**추정 원인**: 
- winning_arrays를 apply() 메서드에서 전체(1,186개)를 전달하는데
- _process_chunk에서 모든 당첨번호와 비교하여 최대 일치 개수를 찾음
- 1,186개 중 하나라도 4개 이상 일치하면 제외

### 2. 효과 없는 필터들
- `ten_section`: 설정이 너무 느슨함
- `arithmetic_sequence`, `geometric_sequence`: 기준값 문제
