## 사용자 질문과 답변

### 1. "그럼 나는 프로그램만 재실행하면 되는거야?"
**답변: 네, 맞습니다!**
- 보너스 번호 수집이 97% 완료 (1150/1187)
- DB에 bonus_number 컬럼 추가 완료
- 프로그램 재실행시 자동으로 보너스 번호 사용

### 2. "db는 자동으로 저장되는거야?"
**답변: 네, 자동 저장됩니다!**
- SQLite DB는 변경사항 즉시 저장
- `data/lotto_numbers.db`에 영구 저장
- 프로그램 재시작해도 데이터 유지

### 3. "그럼 백테스팅은 이제 2등을 제대로 판단해?"
**답변: 네, 이제 제대로 판단합니다!**
- `src/utils/rank_calculator.py` 생성 완료
- 백테스팅 코드에 등수 계산 로직 추가
- 2등 판정: 5개 일치 + 보너스 번호

### 4. "학습은?"
**답변: 학습에는 큰 영향 없습니다!**
- ML 모델은 메인 6개 번호로 학습 (95% 중요도)
- 보너스 번호는 학습에 포함하지 않음
- 기존 학습 성능 그대로 유지

### 5. "패턴은 굳이 2등이 들어갈 필요가 있어?"
**답변: 패턴 분석에는 불필요합니다!**
- 패턴 분석: 메인 6개 번호만으로 충분
- 보너스 번호: 2등 판정용 (5% 중요도)
- 하지만 정확한 수익률 계산에는 필수
