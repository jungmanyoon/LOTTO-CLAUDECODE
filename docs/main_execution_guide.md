# Main.py 실행 가이드

## 🚀 기본 실행 (F5 실행)

### 자동 실행 모드
main.py를 아무 인자 없이 실행하면 자동으로 **--auto-improve** 모드가 활성화됩니다.

```bash
# VSCode에서 F5로 실행 또는
python main.py
```

### 자동으로 실행되는 기능들 (순서대로)

1. **📊 데이터 수집 및 업데이트**
   - 최신 로또 당첨번호 자동 수집
   - 데이터베이스 업데이트

2. **🔍 패턴 분석**
   - 과거 당첨번호 패턴 분석
   - 통계적 특성 추출

3. **🤖 ML/AI 분석** (최적화됨)
   - LSTM 시계열 예측
   - 앙상블 모델 (RF + XGBoost + NN)
   - Monte Carlo 시뮬레이션 (최적화: 2,000회)
   - 베이지안 추론
   - 프랙탈 패턴 분석 (비활성화됨 - 성능 향상)

4. **📈 백테스팅** (최적화됨)
   - 최적화된 백테스팅 프레임워크 사용
   - 모델 캐싱으로 50-70% 속도 향상
   - 병렬 처리로 멀티코어 활용

5. **🔧 필터링 시스템**
   - 17개 필터 자동 적용
   - 동적 필터 관리
   - 적응형 최적화

6. **📋 예측 번호 생성**
   - ML/AI 예측과 필터링 통합
   - 상위 5개 조합 제시

7. **📊 성능 모니터링**
   - 실시간 성능 대시보드
   - 개선 권장사항 제시

8. **🔄 피드백 루프**
   - 향상된 피드백 시스템
   - 자동 모델 개선

## ⚡ 성능 최적화

### 기본 설정 (F5 실행 시)
- **프랙탈 분석**: 비활성화 (42초 절약)
- **백테스팅**: 최적화된 프레임워크 사용
- **병렬 처리**: CPU 코어 수 - 1
- **캐싱**: 자동 활성화

### 예상 실행 시간
- 전체 실행: 약 2-3분
- 백테스팅: 60-80초 (기존 200초)
- ML/AI 분석: 30-40초

## 🛠️ 고급 옵션 (선택사항)

### 특정 기능만 실행
```bash
# ML/AI 분석만
python main.py --ml-only

# 백테스팅 제외
python main.py --skip-backtest

# 필터링만
python main.py --skip-ml --skip-backtest
```

### 성능 설정
```bash
# 프랙탈 분석 포함 (느림)
python main.py --fractal

# 전체 시뮬레이션 (10,000회)
python main.py --full-simulation
```

## 📁 출력 파일

### 자동 생성되는 파일들
1. **예측 결과**: `results/lotto_predictions_[날짜].json`
2. **백테스팅 결과**: `results/backtest_results_optimized_[날짜].json`
3. **성능 보고서**: `results/performance_report_[날짜].txt`
4. **시각화**: 
   - `bayesian_beliefs.png`
   - `fractal_analysis.png` (프랙탈 활성화 시)

### 캐시 파일
- **모델 캐시**: `cache/models/` (자동 관리)
- **예측 캐시**: 메모리 내 (세션 종료 시 초기화)

## ⚠️ 주의사항

1. **첫 실행 시**
   - 모든 모델 학습으로 시간이 더 걸릴 수 있음
   - 캐시 생성 후 다음 실행부터 빨라짐

2. **메모리 사용**
   - 최소 8GB RAM 권장
   - 16GB 이상에서 최적 성능

3. **CPU 사용**
   - 멀티코어 CPU 권장
   - 4코어 이상에서 병렬 처리 효과

## 🔍 문제 해결

### 실행이 느린 경우
1. 캐시 디렉토리 확인 (`cache/models/`)
2. 로그 파일 확인 (`logs/lotto_app.log`)
3. 프랙탈 분석 비활성화 확인

### 메모리 부족
1. 배치 크기 감소
2. 시뮬레이션 횟수 감소
3. 일부 모델 비활성화
