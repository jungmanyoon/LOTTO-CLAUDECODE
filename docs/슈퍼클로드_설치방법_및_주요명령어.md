## 전체 명령어 목록 (총 17개)

### 개발 명령어 (Development Commands)

#### 1. `/build $ARGUMENTS`
- **목적**: 프레임워크 감지를 통한 프로젝트 빌더
- **자동 페르소나**: Frontend, Backend, Architect, Scribe
- **MCP 통합**: Magic (UI 빌드), Context7 (패턴), Sequential (로직)
- **Wave 지원**: ✅ 지원
- **인자**: `[target]`, `@<path>`, `!<command>`, `--<flags>`

#### 2. `/implement $ARGUMENTS`
- **목적**: 지능형 페르소나 활성화를 통한 기능 및 코드 구현
- **자동 페르소나**: Frontend, Backend, Architect, Security (컨텍스트 의존)
- **MCP 통합**: Magic (UI 컴포넌트), Context7 (패턴), Sequential (복잡한 로직)
- **Wave 지원**: ✅ 지원
- **인자**: `[feature-description]`, `--type component|api|service|feature`, `--framework <name>`, `--<flags>`

#### 3. `/design [domain] [flags]`
- **목적**: 디자인 오케스트레이션
- **자동 페르소나**: Architect, Frontend
- **MCP 통합**: Magic, Sequential, Context7
- **Wave 지원**: ✅ 지원

#### 4. `/workflow [operation] [flags]`
- **목적**: 워크플로우 자동화
- **자동 페르소나**: Architect, Analyzer
- **MCP 통합**: Sequential
- **Wave 지원**: ✅ 지원

### 분석 명령어 (Analysis Commands)

#### 5. `/analyze $ARGUMENTS`
- **목적**: 다차원 코드 및 시스템 분석
- **자동 페르소나**: Analyzer, Architect, Security
- **MCP 통합**: Sequential (주요), Context7 (패턴), Magic (UI 분석)
- **Wave 지원**: ✅ 지원
- **인자**: `[target]`, `@<path>`, `!<command>`, `--<flags>`

#### 6. `/troubleshoot [symptoms] [flags]`
- **목적**: 문제 조사
- **자동 페르소나**: Analyzer, QA
- **MCP 통합**: Sequential, Playwright

#### 7. `/explain [topic] [flags]`
- **목적**: 교육적 설명
- **자동 페르소나**: Mentor, Scribe
- **MCP 통합**: Context7, Sequential

### 품질 명령어 (Quality Commands)

#### 8. `/improve [target] [flags]`

- **자동 페르소나**: Refactorer, Performance, Architect, QA
- **MCP 통합**: Sequential (로직), Context7 (패턴), Magic (UI 개선)
- **Wave 지원**: ✅ 지원
- **인자**: `[target]`, `@<path>`, `!<command>`, `--<flags>`

#### 9. `/cleanup [target] [flags]`
- **목적**: 프로젝트 정리 및 기술 부채 감소
- **자동 페르소나**: Refactorer
- **MCP 통합**: Sequential

#### 10. `/test [type] [flags]`
- **목적**: 테스팅 워크플로우
- **자동 페르소나**: QA
- **MCP 통합**: Playwright, Sequential

### 문서화 및 버전 관리 (Documentation & Version Control)

## 명령어 카테고리

- **개발 (Development)**: build, implement, design
- **계획 (Planning)**: workflow, estimate, task
- **분석 (Analysis)**: analyze, troubleshoot, explain
- **품질 (Quality)**: improve, cleanup
- **테스팅 (Testing)**: test
- **문서화 (Documentation)**: document
- **버전 관리 (Version-Control)**: git
- **메타 (Meta)**: index, load, spawn

## Wave 지원 명령어

다단계 오케스트레이션을 위한 Wave Mode를 지원하는 7개 명령어:
- `/analyze`
- `/build`
- `/design`
- `/implement`
- `/improve`
- `/task`
- `/workflow`

## 성능 프로필

- **optimization**: 캐싱과 병렬 실행을 통한 고성능
- **standard**: 적절한 리소스 사용량의 균형잡힌 성능
- **complex**: 포괄적인 분석을 위한 리소스 집약적 처리

## 전체 플래그 목록

### 계획 및 분석 플래그 (Planning & Analysis Flags)

| 플래그 | 목적 | 토큰 사용량 | 자동 활성화 |
|------|---------|-------------|---------------|
| `--plan` | 작업 실행 전 실행 계획 표시 | 최소 | 수동만 |
| `--think` | 다중 파일 분석 및 구조화된 문제 해결 | ~4K 토큰 | 임포트 체인 >5 파일, 크로스 모듈 호출 >10 |
| `--think-hard` | 깊은 아키텍처 분석 | ~10K 토큰 | 시스템 리팩토링, 병목 현상 >3 모듈 |
| `--ultrathink` | 중요 시스템 재설계 분석 | ~32K 토큰 | 레거시 현대화, 중요 취약점 |

### 압축 및 효율성 플래그 (Compression & Efficiency Flags)

| 플래그 | 목적 | 효과 | 자동 활성화 |
|------|---------|--------|---------------|
| `--uc` / `--ultracompressed` | 기호를 사용한 토큰 감소 | 30-50% 감소 | 컨텍스트 사용량 >75% |
| `--answer-only` | 자동화 없이 직접 응답 | 최소 출력 | 수동만 |
| `--validate` | 사전 작업 검증 및 위험 평가 | 안전 검사 | 위험 점수 >0.7 |
| `--safe-mode` | 최대 검증, 보수적 실행 | 위험한 작업 차단 | 리소스 사용량 >85% |
| `--verbose` | 최대 상세 설명 | 높은 토큰 사용량 | 수동만 |

### MCP 서버 제어 플래그 (MCP Server Control Flags)

| 플래그 | 목적 | 서버 | 자동 활성화 |
|------|---------|--------|---------------|
| `--c7` / `--context7` | 라이브러리 문서 조회 | Context7 | 외부 임포트 감지 |
| `--seq` / `--sequential` | 복잡한 다단계 분석 | Sequential | 복잡한 디버깅, --think 플래그 |
| `--magic` | UI 컴포넌트 생성 | Magic | UI 컴포넌트 요청 |
| `--play` / `--playwright` | 브라우저 자동화 및 E2E 테스팅 | Playwright | 테스팅 워크플로우 |
| `--all-mcp` | 모든 MCP 서버 활성화 | 전체 | 복잡도 >0.8 |
| `--no-mcp` | 모든 MCP 서버 비활성화 | 없음 | 수동만 |
| `--no-[server]` | 특정 서버 비활성화 | 다양함 | 수동만 |

### 서브 에이전트 위임 플래그 (Sub-Agent Delegation Flags)

| 플래그 | 목적 | 옵션 | 자동 활성화 |
|------|---------|---------|---------------|
| `--delegate [mode]` | 병렬 처리 활성화 | files/folders/auto | >7 디렉토리 또는 >50 파일 |
| `--concurrency [n]` | 최대 동시 서브 에이전트 | 1-15 (기본: 7) | 동적 할당 |

### Wave 오케스트레이션 플래그 (Wave Orchestration Flags)

| 플래그 | 목적 | 옵션 | 자동 활성화 |
|------|---------|---------|---------------|
| `--wave-mode [mode]` | Wave 오케스트레이션 제어 | auto/force/off | 복잡도 >0.8 AND 파일 >20 |
| `--wave-strategy [type]` | 오케스트레이션 전략 선택 | progressive/systematic/adaptive/enterprise | 프로젝트 특성 기반 |
| `--wave-delegation [type]` | Wave 시스템 위임 | files/folders/tasks | --delegate와 통합 |

### 범위 및 초점 플래그 (Scope & Focus Flags)

| 플래그 | 목적 | 옵션 |

### 반복 개선 플래그 (Iterative Improvement Flags)

| `--interactive` | 반복 간 사용자 확인 | 비활성화 | 수동만 |

## 플래그 우선순위 규칙

1. **안전 우선**: `--safe-mode`가 모든 최적화 플래그를 재정의
2. **명시적 우선**: 사용자 지정 플래그가 자동 활성화를 재정의
3. **사고 계층**: `--ultrathink` > `--think-hard` > `--think`
4. **MCP 제어**: `--no-mcp`가 개별 서버 플래그를 재정의
5. **범위 계층**: system > project > module > file
6. **마지막 페르소나 우선**: 여러 페르소나 플래그 중 마지막 지정이 사용됨
7. **Wave 제어**: off > force > auto
8. **위임**: 명시적 플래그가 자동 감지를 재정의
9. **루프 모드**: 명시적 설정이 자동 감지를 재정의
10. **압축**: `--uc` 자동 활성화가 `--verbose`를 재정의

## 사용 예시

### 기본 명령어 사용
```bash
# 프로젝트 분석
/analyze

# 새 기능 구현
/implement "사용자 인증 기능"

# 코드 품질 개선
/improve --focus performance

# 문서 생성
/document README.md

# 문제 해결
/troubleshoot "로그인 실패 오류"
```

### 플래그 조합 사용
```bash
# 복잡한 분석 - 아키텍트 페르소나와 함께
/analyze --think-hard --persona-architect --scope system

# UI 컴포넌트 생성 - 압축 모드
/build component --magic --uc --persona-frontend

# 안전 모드 프로덕션 배포
/deploy --safe-mode --validate --persona-devops

# 반복적 코드 개선
/improve --loop --iterations 5 --persona-refactorer

# Wave 기반 시스템 최적화
/improve --wave-mode force --wave-strategy progressive --focus performance

# 다국어 문서화
/document --persona-scribe=ja --context7

# 병렬 파일 분석
/analyze --delegate files --concurrency 10

# 특정 MCP 서버 비활성화
/build --no-magic --seq --c7

# 대규모 시스템 리팩토링
/improve --wave-mode auto --wave-strategy systematic --scope system

# E2E 테스트 실행
/test e2e --playwright --persona-qa --validate

# API 구현 - 백엔드 페르소나
/implement "RESTful API" --type api --persona-backend --seq

# 보안 감사
/analyze --focus security --persona-security --ultrathink

# Git 워크플로우
/git commit -m "feat: add authentication" --persona-scribe=en
```

### 고급 사용 예시
```bash
# 복잡한 다중 도메인 분석
/analyze @src/ --wave-mode force --wave-strategy adaptive \
  --persona-architect --all-mcp --scope project

# 엔터프라이즈 수준 보안 감사
/analyze --focus security --wave-mode auto --wave-validation \
  --persona-security --ultrathink --scope system

# 레거시 시스템 현대화
/improve @legacy/ --wave-mode force --wave-strategy enterprise \
  --wave-delegation tasks --persona-architect --think-hard

# 포괄적인 성능 최적화
/improve --focus performance --loop --iterations 10 \
  --delegate auto --concurrency 15 --persona-performance

# 다중 페르소나 협업
/build "e-commerce platform" --persona-architect --persona-frontend \
  --persona-backend --wave-mode auto --all-mcp
```
